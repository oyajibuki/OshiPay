# OshiPay プロジェクト・ルール（AIエージェント用）

このファイルは、AIエージェントが当プロジェクトで作業を開始する際に必ず読み込み、遵守すべきルールを定めたものです。

## 1. ファイル操作と安全管理（必須）
- **バックアップの徹底**: 
    - 既存のファイルを変更・上書き・削除する際は、操作前に必ず `backup/` フォルダ、または同じディレクトリに `.bak` 等の接尾辞を付けて原本を保存すること。
    - 特に **画像、アイコン、メディアファイル（.png, .jpg, .ico 等）** は、一度失うと復元できないため、絶対にバックアップなしで上書きしてはならない。
- **破壊的操作の確認**: 
    - ファイルの削除（rm）や強制移動（mv）を行う前には、必ずその理由をユーザーに明示し、承認を得ること。

## 2. 開発・品質管理
- **アクセスカウンターの維持**: `index.html` などの主要ファイルに埋め込まれているアクセスカウンター（Google Apps Script連携等）は、リファクタリング時に誤って消さないよう注意すること。
- **StreamlitとLPの分離**: `app.py`（Streamlitアプリ）と `docs/`（GitHub Pages用LP）の役割を混同しないこと。LPの修正は必ず `docs/` フォルダ内に対して行う。

## 3. インシデント発生時の対応
- 直ちに作業を停止し、発生した事象、原因、現在の状態を詳細にユーザーへ報告すること。
- データの損失が疑われる場合は、いかなる追加の書き込み操作も行わず、復旧手段を最優先で検討すること。
